# Git
Strumento di versioning, l'obiettivo è quello di supportare lo sviluppo.

Ci sono diversi approcci:
- **Centralized version control** -> esiste un database centrale, sul quale tutti gli utenti lavorano
	- possibilità di disallineamenti nelle versioni a causa di modifiche concorrenti
	- necessita di connessione costante alla rete
- **Distributed access control** -> si parte da una repository di riferimento che viene *clonata*
	- ogni programmatore ha una copia della repo in locale
	- ogni modifica verrà propagata
	- in caso di conflitto di avanzamento di versioni di file sarà necessario conciliarlo
